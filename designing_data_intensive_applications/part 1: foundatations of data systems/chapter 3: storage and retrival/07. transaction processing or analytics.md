# Transaction Processing or Analytics

- The term "transaction" came from a period that writes to dbs were typically related with `commercial transactions`
  - today, even as dbs have expanded beyong this financial scope, the term has stuck

> Transaction: a group of reads and writes that form a logical unit

> Transaction doesn't necessarily mean ACID (atomicity, consistency, isolation, durability) properties. Transaction processing means allowing clients to make low-latency read and writes, as opposed to batch processing jobs, which only run periodically

- Basic access pattern remained similar to processing business transactions

1. look up some small number of records by some key, using an index
2. records are inserted/updated based on user input

- since the applications are interactive, this pattern became known as Online Transaction Process (OLTP)
- Dbs also started being used for analytics, even though their nature is different
  - Instead of few structured rows, they have a long log output and we only need a portion of it, the remaining is noise

- Online Analytic Processing (OLAP)
  - queries that are often written by business analysts to feed into reports to help better business decisions (business intelligence)

| Property           | OLTP                                               | OLAP                                   |
| ------------------ | -------------------------------------------------- | -------------------------------------- |
| Main write pattern | small number of records per query, fetched per key | aggregate over large number of records |
| Main write pattern | random-access, low-latency writes from user input  | bulk import or event stream            |
| Primarily used by  | end user, via web application                      | internal analyst for decision support  |
| Dataset size       | gb to tb                                           | tb to petabytes                        |

- After 1990, there was a trend for companies to stop using the same db for OLTP and OLAP
  - the separate db that analytics was run was called Data warehousing

## Data warehousing

- Data warehouse is a db that analysts can query as much as they want, differently from OLTP, which could cause slower performance
- Data warehouse contains a read-only copy of all the various OLTP systems

1. Data is extracted from OLTP dbs (periodic dump or stream of updates)
2. Transformed into an analysis-friendly schema and cleaned up
3. Loaded into the data warehouse

The process above, of getting data into the data warehouse is called ETL [Extract-Transform-Load]

- Data warehouses exist in almost all large enterprises, but in small companies they aren't used/that useful

- Usually facts are captured as individual events
- A big enterprise [apple, walmart, ebay] may have tens of petabytes of transaction history in its data warehouse, most of which is in fact tables
