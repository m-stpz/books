# Summary

- Data is stored in rows and colums, on sql, at the logical level, but physicially, data is stored in data pages
  - data is stored in data pages [blocks of data with fixed size - usually 8kb]
  - this data lives in disk
  - when we perform a query, sql needs to:
    1. bring it to memory [RAM]
    2. once it's in memory, it traverses every single row
    3. identifies if any row matches the given query

> The operation above is known as **Full Table Scan**

- To improve full table scans, we use indexes
- Data page is the fundamental unit of data storage in sql server
  - data page 8kb
  - table data is saved in a tree-like structure
    - this structure is called b-tree
    - the leaf nodes are what contain actual data
