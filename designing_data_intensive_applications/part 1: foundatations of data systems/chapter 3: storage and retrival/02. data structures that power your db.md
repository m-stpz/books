# Data structures that power your db

- Consider the world's simplest db, implemented in TS
  - A basic key/value storage system

```ts
import * as fs from "fs";

class TinyDB<T> {
  // O(1) look time with Map
  private data: Map<string, T>;
  private filePath: string;

  constructor(filePath: string) {
    this.filePath = filePath;

    // load existing data or start fresh
    if (fs.existsSync(filePath)) {
      const raw = fs.readFileSync(filePath, "utf-8");
      this.data = new Map(Object.entries(JSON.parse(raw)));
    } else {
      this.data = new Map();
    }
  }

  // save the current state to the disk
  private save() {
    const obj = Object.fromEntries(this.data);
    // serialize the whole map into a JSON string and overwrite the local file
    fs.writeFileSync(this.filePath, JSON.stringify(obj, null, 2));
  }

  set(key: string, value: T): void {
    this.data.set(key, value);
    this.save(); // save the result on mutation
  }

  get(key: string): T | undefined {
    return this.data.get(key);
  }

  delete(key: string): boolean {
    const result = this.data.delete(key);
    this.save(); // save the result on mutation
    return result;
  }
}

interface User {
  //...
}

const db = new TinyDB<User>(".database.json");

db.set("user_1", { name: "Alice", isCool: true });
console.log(db.get("user_1"));
```

- Appending something to a file is usually rather efficient
  - This is log

> Even though log is used to refer to application logs (where application outputs text to describe what's going on), in this context it means: an append-only sequence of records (human-readable or not)

- This is the core principles of dbs
  - of course, with much more xomplex logic/controls for:
    - concurrency control
    - reclaiming disk space
    - handling error
    - handling partially written records

- Our db has a map, instead of an array.
  - Look up O(1) instead of O(n)
  - Indexing allows efficient data storage
    - indexing: keeping additional metadata on the side
      - acts as a sign post and helps locating the data more easily (e.g. key-value pair)
    - For writes, it's difficult simply appending to the end of some data structure. Some, any kind of indexing will slow writing down, however, it can dramatically increase read-time

## Hash indexes

- Key-value stores are quite similar to the dictionary/object type in many languages
- The simplest index strategy is: keep an in-memory hash map of where every key is mapped to a byte offset in the data file

| key     | byte offset                                                           |
| ------- | --------------------------------------------------------------------- |
| value 1 | 0 // this is the range where this value [e.g. an object] lives (0-64) |
| value 2 | 64                                                                    |

- Given that we only append to a file, how do we avoid running out of space?
  - Since, in our current conceptualization, we're just appending, how do we deal with repeated values?
    - Break the log into segments of a certain size by closing the segment once it achieves a certain size. This is called compaction
    - I allocate "5 sizes" for this segment

```ts
interface LogEntry {
  key: string;
  value: any;
}

function compact(segments: LogEntry[]): LogEntry[] {
  const latestValues = new Map<string, any>(); // initialize an empty map to store the keys we've seen while moving "backwards"

  // iterate from most recent to oldest
  for (let i = segments.length - 1; i >= 0; i--) {
    const entry = segments[i];

    /* 
        most recent entry 10 => we only care about this one
                ... the others we eliminate
                    entry 5
                    entry 4
        oldest      entry 1
    */

    // if we haven't seen this key yet, it's the most recent one (only add the first seen key -> the others are already "stale")
    if (!latestValues.has(entry.key)) {
      latestValues.set(entry.key, entry.value);
    }
  }

  // convert the map into an array of entries
  // these are unique, meaning we compacted the entries
  return Array.from(latestValues.entries()).map(([key, value]) => ({
    key,
    value,
  }));
}

const logHistory: LogEntry[] = [
  { key: "user_1", value: { name: "Alice", status: "online" } }, // Entry 0 (Oldest)
  { key: "user_2", value: { name: "Bob", status: "offline" } }, // Entry 1
  { key: "user_1", value: { name: "Alice", status: "away" } }, // Entry 2 (Update)
  { key: "user_3", value: { name: "Charlie", status: "online" } }, // Entry 3
  { key: "user_1", value: { name: "Alicia", status: "active" } }, // Entry 4 (Final Update)
];

console.log(compact(logHistory));
```
