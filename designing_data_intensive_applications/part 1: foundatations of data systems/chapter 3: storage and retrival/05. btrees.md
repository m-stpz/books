# B-Tress

- Log-structured indexes, which we've seen previously, are gaining acceptance, but aren't the most common type of indexing
- The most widely used indexing structure is a B-tree
- B-trees were introduced in 1970 and remain the standard index implementation in all relational dbs, and many nonrelational use them too
- Log-structured indexes:
  1. break the db into variable-size segments
  2. write segments sequentially

- B-trees:
  1. break the db into fixed-size blocks/pages (usually 4kb in size)
  2. read or write on page at a time
     2.1 pages are identified using an address or location
     2.2. pages can "refer" to one another (similar to a pointer, but in disk instead of memory)

```
            | ref |100 | ref| 200| ref | 300|
                    /              \
| ref |111 | ref| 140| ref | 160|   \
                                     \
                                | ref |220 | ref| 230| ref | 270|
```

- One page is the root
  - references the child pages
  - each child is responsible for a continuous range of keys
  - keys between the references determine the range boundary between the keys

- A four-level tree of 4KB pages with a branching factor of 500 can store up to 250TB
  - branching factor = number of references to child pages in one page
