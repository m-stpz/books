# Summary Chapter 1

- An application must meet various requirements to be useful
- Functional requirements: what it should do
  - allowing data to be stored, retrieved, searched, and processed in various ways
- Nonfunctional requirements: how it should do
  - "ities": security, reliability, compliance, scalability, maintainability

- Reliability: making the systems continue working, even in adverse situations
  - Faults can be hardware, software, human
  - Fault-tolerance mechanisms

- Scalability: keeping good performance, even when load increases
  - Flexible or manual
  - Definining load
    - Percentiles
    - Throughput
    - Only data throughput isn't enough, each application has a specific need
      - A system that has 100k req/sec with 1kb is different from one that handles 3 req/min with 2gb
    - Scale system horizontally, vertically, or hybrid

- Maintaibility
  - Live should be easy for operations
  - Operability, Simplicity, Maintainability
  - Good abstractions help
  - Visibility into the systems health and ways to manage it
