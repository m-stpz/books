# Data models and query languages

- Data models can be the most important part of developing software. They have a great impact in:
  - How the software is written
  - How we think about the problem we're solving

- Most applications are built by layering one data model on top of another
- For each layer, the key question is:

> How is it represented in terms of the next-lower layer?

1. Real world: As an application developer, you look at the real world (people, organizations, good, actions, money flows, sensors, needs, etc) and model it in terms of objects and data structures, and APIs to manipulate them. Structures specific to your application
2. Data model: When we want to store the data structure, express them in terms of a general-purpose data model (JSON, XML, tables in a relational db, graph model, etc)
3. Representing data into bytes: The engineers who built the db software figured out a way to represent that data (JSON/XML/relational/graph) in terms of bytes in memory, on disk, or a network. This representation allows data to be queried, searched, manipulated, and processes somehow
4. Bytes into physical phenomena: On yet lower levels, hardware engineers have found a way to represent bytes into electrical currents, pulses of light, magnetic fields, etc

- In complex applications there may be more intermediate levels (e.g. APIs built on APIs), but the idea is the same

> Each layer "hides" the complexity of the layer below by providing a clean data model
