# Query languages for data

- SQL is a declarative query language, not imperative
  - imperative: describe the steps. You say "how"
    - tells the computer to perform certain operations in certain order
    - stepping through the code, line by line, evaluating conditions, updating variables, and deciding whether to go around the loop on more time
    - harder to parallelize
  - declarative: describe the pattern. You say "what"
    - what conditions the results must meet and how the data should be transformed (e.g., sorted, grouped, aggregated)
    - you don't say "how" to achieve this goal
    - up to db systems to decide how
    - typically more concise and easier to work with
      - hides the implementation better
      - by hiding the inner implementations better, we can improve the inner workings without changing the way we interact with the system
    - easier to parallelize

```ts
// imperative code = most commonly used programming languages are imperative

function getAnimalType(animals, type) {
  const results = [];

  for (const animal of animals) {
    if (animal.type === type) {
      result.push(animal);
    }
  }

  return results;
}
```

## Declarative queries on the web

- There are advantages to declarative languages go beyond dbs
- Compare the difference: situation -> the title of the currently selected page to have a blue background

declarative: e.g. css

```css
li.selected > p {
  background-color: bluel;
}
```

imperative: e.g. javascript

- this code compared with the above, it's:
  - much harder to read
  - doesn't "unselect" as easily as the css and its automatic rule removal
  - to use the latest features, you must update the code, differently from css, which hides its inner workings

```js
const liElements = document.getElementsByTagName("li");

for (let i = 0; i < liElements.length; i++) {
  if (liElements[i].className === "selected") {
    const children = liElements[i].childNodes;

    for (let j = 0; j < children.length; j++) {
      const child = children[j];

      if (child.NodeType === Node.ELEMENT_NODE && child.tagName === "P") {
        child.setAttribute("style", "background-color:blue");
      }
    }
  }
}
```

- Declarative query languages turned out much better than imperative query APIs

## MapReduce Querying

- MapReduce is a programming model for processing large amounts of data in bulk across many machines
  - Popularized by google
- MapReduce isn't declarative, nor imperative. It's somewhere in between
- Situation: You want to increase a counter every time you encounter given ocurrence per time frame
  - Sharks spotted at the beach per month
  - Customers signing up to newsletters per week
  - etc

```sql
SELECT date_trunc('month', observation_timestamp) AS observation_month,
       sum(num_animals) AS total_animals
FROM observations
WHERE family = 'Sharks'
GROUP BY observation_month;
```

```js
db.observations.mapReduce(
  function map(){ // 2. map is called once for every document that matches the query
    const year = this.observationTimeStamp.getFullYear();
    const month = this.observationTimeStamp.getMonth() + 1;
    emit(`${year} - ${month}: ${this.numAnimals}`)
  }

  function reduce(key, values){ // 3. the key-value pairs emitted by map are grouped by key
    return Array.sum(values);
  }

  {
    query: {family:"Sharks"}, // 1. filter to consider [categort]
    out:"monthlySharkReports"
  }
)
```

- A NoSQL system may find itself accidentally reinventing SQL, albeit in disguise
