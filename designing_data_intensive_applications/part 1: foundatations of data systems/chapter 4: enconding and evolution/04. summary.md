## Summary

- We looked on how to transform data into bytes (network and disk)
- These encodings not only increase efficiency, but also change the way we architect/evolve applications
- Rolling upgrades: updating nodes gradually
  - It's important that our data is both backward- and forward-compatible
    - backward compatibility (new code handles old code)
    - forward compatibility (old code handles new code)

- Dataflows & encoding
  - Dbs: process writing to the db encode the data and process reading it, decodes it
  - RPC and REST: client encodes request, server decodes the request and encodes a response, client decodes the response
  - Message queues: encoded by the sender and decoded by the recipient
